## Directory Purposes

**apps/backend/src/db/schema/:**
- Purpose: Define all database table structures, enums, relations
- Contains: Table definitions using Drizzle ORM primitives
- Key files:
  - `plant.schema.ts` - Plant table with 40+ columns (classification, characteristics, sowing info)
  - `seed.schema.ts` - Seed varieties linked to plants
  - `auth-schema.ts` - User table from Better-Auth
- Pattern: Each file exports `table`, `tableRelations`, and TypeScript types (`Plant`, `NewPlant`)

**apps/backend/src/services/:**
- Purpose: Business logic and database query building
- Contains: Service objects with methods for CRUD operations
- Key files:
  - `plant.service.ts` - `plantService` with pagination, filtering, sorting
  - `seeds.service.ts` - `seedsService` for seed management
- Pattern: Exported as named object with async methods, no class syntax

**apps/backend/src/routes/:**
- Purpose: HTTP endpoint definitions and validation
- Contains: Hono route groups with middleware and handlers
- Key files:
  - `plant.routes.ts` - GET/POST/PATCH/DELETE /api/plant endpoints
  - `seeds.routes.ts` - Nested routes for /api/plant/:plantId/seeds
  - `auth.routes.ts` - Authentication endpoints (login, signup, logout)
- Pattern: Each file exports `new Hono()` with `.use()` middleware, then `.get()/.post()/.patch()/.delete()`

**apps/backend/src/middleware/:**
- Purpose: Middleware for authentication, logging, error handling
- Contains: Reusable middleware functions
- Key files:
  - `auth.middleware.ts` - `requireAuth` verifies session and injects userId into context
- Pattern: Use Hono's `createMiddleware<{ Variables }>()` for typed context

**apps/frontend/src/routes/:**
- Purpose: File-based routing using TanStack Router conventions
- Contains: Page components that export `Route` definition
- Naming Conventions:
  - `index.tsx` - List/default page
  - `new.tsx` - Create page
  - `$id.index.tsx` - Detail page
  - `$id.edit.tsx` - Edit page
  - `_authenticated/` - Protected route group (requires auth)
  - `$seedId.edit.tsx` - Dynamic segment
- Pattern: Each file exports `const Route = createFileRoute(...)()`

**apps/frontend/src/hooks/:**
- Purpose: React hooks for data fetching and mutations via TanStack Query
- Contains: Custom hooks wrapping `useQuery` and `useMutation`
- Key files:
  - `usePlants.ts` - Hooks: `usePlants()`, `usePlant()`, `useCreatePlant()`, `useUpdatePlant()`, `useDeletePlant()`
  - `useSeeds.ts` - Hooks for seed operations
- Pattern: Hierarchical query keys, `onSuccess` cache invalidation

**apps/frontend/src/services/:**
- Purpose: API client methods using the `request<T>()` helper
- Contains: Methods that call backend endpoints
- Key files:
  - `plants.service.ts` - `plantsApi` object with `getAll()`, `getById()`, `create()`, `update()`, `delete()`
  - `seeds.service.ts` - `seedsApi` for seed operations
- Pattern: Named exports (`plantsApi`, `seedsApi`), all use `request<T>()` helper

**apps/frontend/src/@types/:**
- Purpose: TypeScript type definitions for frontend consumption
- Contains: Types inferred from or aligned with backend schemas
- Key files:
  - `api.types.tsx` - `ApiResponse<T>`, `PaginatedResponse<T>`
  - `plant.types.tsx` - `Plant`, `CreatePlantInput`, `PlantsQueryParams`
  - `seed.types.tsx` - Seed types
- Pattern: Types match backend schemas, used in hooks and services

**apps/frontend/src/components/plant/:**
- Purpose: Reusable components for plant domain
- Contains: Form components and UI elements
- Key files:
  - `plant-form.tsx` - Form with React Hook Form + Zod validation
  - `plant-seed-selection.tsx` - Seed variety picker
- Pattern: Component exports use PascalCase (`export function PlantForm`)

**packages/shared-types/src/schemas/:**
- Purpose: Single source of truth for validation and types
- Contains: Zod schemas for both backend and frontend
- Key files:
  - `plant.schema.ts` - `createPlantSchema`, `plantSchema`, type inference
  - `seed.schema.ts` - Seed schemas
  - `common.schema.ts` - Shared utilities (enums, helpers)
- Pattern: Schemas drive validation, types inferred via `z.infer<typeof schema>`

## Key File Locations

**Entry Points:**
- Backend API: `apps/backend/src/app.ts` (Hono app definition)
- Backend Runtime: `apps/backend/src/index.ts` (server startup)
- Frontend Root: `apps/frontend/src/routes/__root.tsx` (HTML + layout)
- Frontend Router: `apps/frontend/src/router.tsx` (TanStack Router setup)

**Configuration:**
- Backend DB: `apps/backend/drizzle.config.ts`
- Backend Vite: `apps/backend/vitest.config.ts`
- Frontend Vite: `apps/frontend/vite.config.ts`
- Workspace: `pnpm-workspace.yaml`, `turbo.json`

**Core Logic:**
- Plant CRUD: `apps/backend/src/routes/plant.routes.ts` + `apps/backend/src/services/plant.service.ts`
- Seed CRUD: `apps/backend/src/routes/seeds.routes.ts` + `apps/backend/src/services/seeds.service.ts`
- Plant UI: `apps/frontend/src/routes/_authenticated/plants/` + `apps/frontend/src/components/plant/`

**Testing:**
- Backend unit tests: `apps/backend/src/**/*.test.ts` (co-located)
- Test config: `apps/backend/vitest.config.ts`

## Naming Conventions

**Files:**
- Backend routes: `plant.routes.ts` (singular, kebab-case)
- Backend services: `plant.service.ts` (singular)
- Backend schemas: `plant.schema.ts` (singular)
- Frontend components: `plant-form.tsx` (kebab-case)
- Frontend hooks: `usePlants.ts` (camelCase, no kebab)
- Frontend services: `plants.service.ts` (camelCase)
- Frontend schemas: `plant.schema.ts` (singular, kebab-case)
- Frontend types: `plant.types.tsx` (singular, kebab-case)

**Directories:**
- Backend: `src/db/`, `src/routes/`, `src/services/`, `src/middleware/`, `src/lib/`, `src/utils/`
- Frontend: `src/routes/`, `src/components/`, `src/hooks/`, `src/services/`, `src/lib/`, `src/@types/`, `src/schemas/`, `src/data/`
- Protected routes: `_authenticated/` (underscore prefix for route groups)
- Nested resources: `plants/$id.seeds/$seedId.edit.tsx`

**Exports:**
- Backend: Named object exports (`plantService`, `plantRoutes`)
- Frontend components: Function exports with PascalCase (`export function PlantForm()`)
- Frontend services: Named object exports (`plantsApi`, `seedsApi`)
- Frontend hooks: Named function exports (`export function usePlants()`)

## Where to Add New Code

**New Feature (e.g., plant companion plants):**
1. Add database table in `apps/backend/src/db/schema/companion.schema.ts`
2. Add relations in plant.schema.ts
3. Create service: `apps/backend/src/services/companion.service.ts`
4. Create routes: `apps/backend/src/routes/companion.routes.ts`
5. Mount in `apps/backend/src/app.ts` via `app.route("/api/companion", companionRoutes)`
6. Add shared schema: `packages/shared-types/src/schemas/companion.schema.ts`
7. Create hook: `apps/frontend/src/hooks/useCompanions.ts`
8. Create service: `apps/frontend/src/services/companions.service.ts`
9. Create components: `apps/frontend/src/components/companion/` folder
10. Create routes: `apps/frontend/src/routes/_authenticated/companions/`

**New Component/Module:**
- Implementation: `apps/frontend/src/components/{domain}/{name}.tsx`
- Test: `apps/frontend/src/components/{domain}/{name}.test.tsx` (if complex)
- Usage: Import directly, no barrel files used

**Utilities/Helpers:**
- Shared helpers: `apps/frontend/src/lib/utils.ts`
- Backend helpers: `apps/backend/src/utils/` folder (one file per function category)
- Frontend types: `apps/frontend/src/@types/`

**New Route:**
- Public routes: `apps/frontend/src/routes/` (alongside index.tsx, login.tsx)
- Protected routes: `apps/frontend/src/routes/_authenticated/{feature}/`
- Pattern: Use TanStack Router file conventions (index.tsx, new.tsx, $id.index.tsx, $id.edit.tsx)

## Special Directories

**apps/backend/src/db/migrations/:**
- Purpose: Version-controlled database migration scripts
- Generated: Yes, via `pnpm drizzle-kit generate` from schema changes
- Committed: Yes, to version control

**apps/frontend/src/data/:**
- Purpose: Static data and mock fixtures (not for fetched data)
- Generated: No
- Committed: Yes
- Example: `demo.punk-songs.ts` (demo data for TanStack examples)

**apps/frontend/.tanstack/:**
- Purpose: TanStack Start build artifacts and generated code
- Generated: Yes, during build
- Committed: No (in .gitignore)

**packages/shared-types/src/schemas/:**
- Purpose: Central validation schemas imported by backend AND frontend
- Generated: No, hand-written
- Committed: Yes
- Import pattern:
  - Backend: `import { createPlantSchema } from "@happyseeds/shared-types"`
  - Frontend: Same import works due to Turborepo path alias

**apps/backend/src/db/migrations/meta/:**
- Purpose: Metadata about migrations (journal)
- Generated: Yes
- Committed: No (migrations themselves are committed, not meta)
