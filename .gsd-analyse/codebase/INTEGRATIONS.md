## Data Storage

**Databases:**
- PostgreSQL
  - Connection: `DATABASE_URL` environment variable (required)
  - Client: `postgres` npm package (v3.4.5)
  - ORM: Drizzle ORM (v0.45.1)
  - Dialect: PostgreSQL
  - Schema location: `apps/backend/src/db/schema/`
  - Migrations: `apps/backend/src/db/migrations/` (generated by drizzle-kit)

**File Storage:**
- Local filesystem only
- Image URLs stored as text in database (`imageUrl` field in plants table)
- No cloud storage integration

**Caching:**
- None (no Redis, Memcached, or similar)
- TanStack Query handles client-side caching via memory

## Authentication & Identity

**Auth Provider:**
- Custom (self-hosted)
  - Implementation: better-auth 1.4.17
  - Method: Email/password authentication
  - Minimum password length: 8 characters
  - Session expiry: 7 days
  - Session update age: 1 day
  - Location: `apps/backend/src/lib/auth.ts`

**Session Management:**
- Session-based with PostgreSQL storage
- Tables: `users`, `sessions`, `accounts`, `verifications` (managed by better-auth)
- Cookie-based persistence with secure credentials flag
- Backend uses Drizzle adapter for better-auth
- Frontend uses `better-auth/react` client for session management

**CORS & Origins:**
- Trusted origins configured via environment:
  - Frontend: `FRONTEND_URL` or `http://localhost:3000`
  - Backend: `BACKEND_URL` or `http://localhost:3001`
- Credentials (cookies) enabled for cross-origin requests

## Database Schema Overview

**User Management (via better-auth):**
- `users` - User profiles (id, name, email, emailVerified, image, timestamps)
- `sessions` - Active sessions (id, token, expiresAt, userId, IP, user agent)
- `accounts` - OAuth/auth provider accounts (reserved for future OAuth integration)
- `verifications` - Email verification tokens

**Application Data:**
- `plants` - User plant catalog
  - Fields: id (UUID), userId (FK), name, latinName, category, description
  - Sowing: sowingDepthMm, sowingSpacingCm
  - Growth: germinationDays (min/max), growthDays (min/max)
  - Requirements: sunRequirement, waterRequirement
  - Metadata: notes, imageUrl, timestamps
  - Enums: plantCategory, sunRequirement, waterRequirement

- `seeds` - Seed inventory
  - Fields: id (UUID), plantId (FK), userId (FK), brand, quantity
  - Dates: acquisitionDate, expirationDate
  - Metadata: acquisitionType (harvest/purchase/gift/unknown), notes, timestamps
  - Enums: acquisitionType

## API Endpoints

**Authentication (better-auth managed):**
- `POST /api/auth/sign-up` - User registration
- `POST /api/auth/sign-in` - User login
- `POST /api/auth/sign-out` - User logout
- `GET /api/auth/session` - Get current session
- `POST /api/auth/revoke-sessions` - Revoke sessions
- Additional endpoints auto-managed by better-auth

**Health & Status:**
- `GET /api/health` - Server health check
- `GET /api/db-test` - Database connectivity test

**Plants CRUD:**
- `GET /api/plants` - List plants with pagination/filtering
- `GET /api/plants/:id` - Get single plant
- `POST /api/plants` - Create plant
- `PATCH /api/plants/:id` - Update plant
- `DELETE /api/plants/:id` - Delete plant

**Seeds CRUD:**
- Routes defined in `apps/backend/src/routes/seeds.routes.ts` (not fully detailed)

## Frontend API Integration

**API Client Location:** `apps/frontend/src/lib/api-client.ts`

**API Response Format:**
```typescript
Success: { success: true; data: T; }
Error: { success: false; error: { code: string; message: string; details?: Record<string, string>; }; }
Paginated: { success: true; data: T[]; pagination: { page, limit, total, totalPages }; }
```

**Request Configuration:**
- Base URL: `VITE_API_URL` or `http://localhost:3001`
- Credentials: `include` (sends cookies)
- Content-Type: `application/json`
- Error handling: Custom ApiError class with code, message, status

**Authentication Client Location:** `apps/frontend/src/lib/auth-client.ts`
- Built-in methods: signIn, signUp, signOut, revokeSessions, useSession
- Uses better-auth React client
- Credentials enabled for cross-origin requests
